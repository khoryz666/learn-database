Data redundancy and Update Anomalies
◼A relation should contains minimal data redundancy and 
allows users to insert, delete, and update rows without 
causing data inconsistencies
◼The following anomalies to be avoided:
◼Insertion Anomaly –adding new rows forces user to 
create duplicate data
◼Update or Modification Anomaly –changing data in 
a row forces changes to other rows because of 
duplication
◼Deletion Anomaly –deleting rows may cause a loss 
of data that would be needed for other future rows
