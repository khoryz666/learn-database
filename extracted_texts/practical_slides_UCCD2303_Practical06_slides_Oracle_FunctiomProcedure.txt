Stored program units
◼Stored program units can be compiled 
separately and stored permanently in an 
Oracle database, ready to be executed. 
◼Stored program units are stored in parsed, 
compiled form. So, when called, they are 
loaded and passed to the PL/SQL engine 
immediately.

Creating a Stored Program Unit: Function
CREATE OR REPLACE FUNCTION age
(date_of_birth IN DATE)
RETURN NUMBER
IS
current_age NUMBER;
BEGIN
current_age:=TRUNC((SYSDATE –date_of_birth)/365.25);
RETURN current_age;
END;

Packages
◼A collection of procedures, functions, variables, and 
SQL statements that are grouped together and 
stored as a single program unit.
◼Consist of two parts:
❑Specification –declares all public constructs of the 
package.
❑Body –defines all constructs (public and private) of the 
package, and so implements the specification . 
|-UCCD2303/UCCD2203 -| 23

Packages
◼Oracle performs the following steps when a 
procedure or package is created:
❑It compiles the procedure or package.
❑It stores the compiled code in memory.
❑It stores the procedure or package in the database.

Creating Database Triggers to Leave an Audit Trail for 
the FOODORDERS Table
|-UCCD2303/UCCD2203 -| 30SQL> CREATE TABLE foodorders
( order_id number(5), quantity number(4),

CREATE TABLE foodorders_audit
( order_id number(5),  quantity number(4),
cost_per_item number(6,2),
total_cost AS (quantity*cost_per_item),
create_date date,
CONSTRAINT order_id_unique UNIQUE (order_id)
);

Creating a Row -level Trigger
SQL> INSERT INTO foodorders (order_id , quantity, 
cost_per_item ) values (1,2,10);
To create and test the row -level trigger: before_insert
SQL>CREATE OR REPLACE TRIGGER orders_before_insert
BEFORE INSERT
ON foodorders
FOR EACH ROW
BEGIN
--Update create_date field to current system date
:new.create_date := sysdate;
END;

Disabling and Dropping Triggers
◼Disable trigger
❑Trigger still exists in user’s database schema 
❑No longer fires when triggering event occurs
❑Syntax:
ALTER TRIGGER trigger_name [ENABLE | DISABLE];
SQL> ALTER TRIGGER order_before_insert DISABLE;
SQL> ALTER TRIGGER order_before_insert ENABLE;
◼Drop trigger when not needed:
DROP TRIGGER trigger_name ;
SQL> DROP TRIGGER order_before_insert ;
